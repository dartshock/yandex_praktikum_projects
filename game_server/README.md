# Game Server: "Dog Story"

Асинхронный игровой сервер на C++ для многопользовательской игры.
Игроки управляют собаками, которые перемещаются по карте, собирают трофеи и сдают их в офисах для получения очков.

## Технический обзор

### Ключевые компоненты:
- **Network** — асинхронный HTTP сервер на Boost.Beast
- **Application** — use cases, управление игроками и токенами
- **Model** — игровая логика (карты, собаки, трофеи, коллизии)
- **Persistence** — PostgreSQL через пул соединений + файловая сериализация

###  Основные технологии
- **Boost.Asio** — асинхронность
- **Boost.Beast** — HTTP
- **Boost.JSON** — парсинг
- **Boost.Log** — логирование
- **Boost.Serialization** — сохранение состояния игры
- **PostgreSQL + libpqxx** — хранение рекордов

### Возможности
- **HTTP API** — полный набор REST-like эндпоинтов
- **Игровая логика** — движение по дорогам, сбор трофеев, система коллизий
- **Генерация трофеев** — вероятностная модель появления предметов
- **Сохранение данных** — PostgreSQL для рекордов + файловая сериализация состояния
- **Асинхронность** — высокая производительность через Boost.Asio
- **Логирование** — структурированные JSON-логи
- **Многопоточность** — пул потоков и пул соединений с БД

## API Endpoints

### Карты
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/api/v1/maps` | Список всех карт |
| GET | `/api/v1/maps/{id}` | Детальная информация о карте |
### Игра
| Метод | Эндпоинт | Тело запроса | Описание |
|-------|----------|--------------|----------|
| POST | `/api/v1/game/join` | `{"userName": "<string>", "mapId": "<string>"}` | Войти в игру |
| GET | `/api/v1/game/players` | — | Список игроков в сессии |
| GET | `/api/v1/game/state` | — | Текущее состояние игры |
| POST | `/api/v1/game/player/action` | `{"move": "U/D/L/R/"}` | Движение игрока |
| POST | `/api/v1/game/tick` | `{"timeDelta": <ms>}` | Тик времени |
| GET | `/api/v1/game/records` | — | Таблица рекордов |

## Сборка и запуск сервера

### Требования
- CMake 3.11+
- Conan 1.66.0+
- Компилятор с C++20
- PostgreSQL (для рекордов)

* Инструкция тестировалась на Conan 1.66.0.

### Linux
```
# mkdir release && cd release
# conan install .. --build=missing -s build_type=Release -s compiler.libcxx=libstdc++11
# cmake .. -DCMAKE_BUILD_TYPE=Release
# cmake --build .
```

## Сборка под Windows
```
# mkdir build && cd build
# conan install .. --build=missing -s build_type=Release
# cmake --build . --config Release
```

## Запуск сервера
Перед запуском необходимо настроить подключение к базе данных PostgreSQL через переменную окружения `GAME_DB_URL`.

```
export GAME_DB_URL="postgres://username:password@host:port/database_name"
./game_server -c /path_to_config.json -w /path_to_static_files/ [options]
```

### Обязательные аргументы:
| Параметр | Описание |
|----------|----------|
|`-c, --config-file <file>`|путь к JSON-файлу с конфигурацией карт |
|`-w, --www-root <dir>`|путь к директории со статическими файлами (HTML, CSS, JS).

### Опциональные аргументы:
| Параметр | Описание |
|----------|----------|
|`--state-file <file>`|путь к файлу для сохранения/восстановления состояния игры.
|`--save-state-period <ms>`|периодичность автосохранения состояния в миллисекундах.
|`--tick-period <ms>`|включает автоматический режим обработки игрового времени с заданным интервалом. Если не указан, используется режим обработки по запросу (/api/v1/game/tick).
|`--randomize-spawn-points`|включает случайный спавн собак на дорогах карты. По умолчанию спавн происходит в начале первой дороги.
|`-h, --help`|показать справку.
